## はじめに

2026年2月25日、Anthropic が **Claude Code Remote Control** を発表しました。

すでに「使い方」の記事は出ていますが、この記事では一歩踏み込んで**MCP（Model Context Protocol）との組み合わせ**に焦点を当てます。

Remote Control の真の価値は「スマホから操作できる」ことではなく、**ローカル環境のMCPサーバーにモバイルからアクセスできる**ことです。

私は副業で iOS アプリ「ふくログ」を開発しながら、Notion で事業KPIを管理しています。この記事では、実際に Remote Control × Notion MCP で以下を実現した手順を紹介します。

- iPhoneからNotionの事業KPIを更新
- 移動中にClaude Codeでバグ修正を指示
- ローカルのスクリプト・環境変数・MCPサーバーをそのまま活用

## Remote Control とは（30秒で理解）

```
┌─────────────┐      HTTPS      ┌──────────────┐      HTTPS      ┌─────────────┐
│  iPhone /    │ ◄──────────── │  Anthropic    │ ──────────────► │  自宅PC      │
│  ブラウザ     │                │  API          │                │  Claude Code │
└─────────────┘                └──────────────┘                └──────┬──────┘
                                                                      │
                                                               ローカル環境
                                                               ├── ファイルシステム
                                                               ├── 環境変数(.env)
                                                               ├── MCP サーバー
                                                               └── Git / npm 等
```

ポイント：
- **セッションはローカルで動き続ける**（クラウドに移動しない）
- スマホはその「窓」として機能するだけ
- MCP サーバー、環境変数、ファイルシステムすべてそのまま使える
- アウトバウンド HTTPS のみ（インバウンドポート不要）

## なぜ MCP × Remote Control が強いのか

Claude Code on the Web（ブラウザ版）でも開発はできます。しかし決定的な違いがあります。

| 観点 | Remote Control | Claude Code on the Web |
|------|---------------|----------------------|
| ローカルファイル | アクセス可能 | 不可 |
| **MCPサーバー** | **利用可能** | **不可** |
| 環境変数(.env) | 利用可能 | 不可 |
| Git 操作 | 可能 | 制限あり |

MCP が使えるかどうかで、できることが根本的に変わります。

たとえば私の環境では Notion MCP が設定されていて、Claude Code から直接 Notion のデータベースを読み書きできます。Remote Control なら、この MCP 接続をスマホから使えるのです。

## 前提条件

| 項目 | 必要なもの |
|------|-----------|
| Claude Code | v2.1.52 以上 |
| プラン | Pro ($20/月) または Max ($100〜200/月) |
| 認証 | `claude` コマンドで `/login` 済み |
| MCP サーバー | 設定済み（この記事では Notion MCP を使用） |
| モバイル | Claude iOS/Android アプリ |

## セットアップ

### 1. Claude Code のアップデート

```bash
npm update -g @anthropic-ai/claude-code
claude --version  # v2.1.52 以上であることを確認
```

### 2. MCP サーバーの設定確認

私の環境では、Claude Code の設定ファイル（`~/.claude/settings.json` または プロジェクトの `.claude/settings.json`）に Notion MCP が設定されています。

```json
{
  "mcpServers": {
    "Notion": {
      "type": "url",
      "url": "https://mcp.composio.dev/notion/xxxxx"
    }
  }
}
```

MCP サーバーの種類は何でもOKです（Notion、Slack、GitHub、Supabase、PostgreSQL など）。Remote Control ではこれらがすべてモバイルから利用可能になります。

### 3. Remote Control の起動

新規セッションで開始する場合：

```bash
claude rc
```

既存セッションから切り替える場合は、Claude Code 内で：

```
/rc
```

起動すると、ターミナルに **セッション URL** と **QR コード** が表示されます。

### 4. スマホから接続

**方法A: QRコード（おすすめ）**
1. ターミナルでスペースキーを押して QR コードを表示
2. iPhone のカメラで QR コードをスキャン
3. Claude アプリが自動で開き、セッションに接続

**方法B: セッション一覧**
1. Claude iOS アプリを開く
2. セッション一覧に Remote Control セッションが表示される
3. タップして接続

![Claude iOSアプリのセッション一覧](/images/rc-session-list.jpg)
*「進行中」の下に「Interactive session」として表示される。タップするだけで接続完了。*

## 実践: iPhone から Notion の事業KPIを更新する

ここからが本題です。Remote Control × Notion MCP の実践例を紹介します。

### ユースケース1: 移動中にKPIダッシュボードを確認・更新

私は Notion に「事業KPIトラッカー」というデータベースを持っています。月次の収益、フォロワー数、Go/No-Go判定などを管理しています。

Remote Control で接続後、iPhone の Claude アプリから：

```
今月のKPIトラッカーを確認して、Zennフォロワー数を最新の値に更新して
```

実際に iPhone から操作した画面がこちらです：

![iPhoneからNotion MCPでKPIトラッカーを表示](/images/rc-notion-kpi.jpg)
*iPhoneのClaude アプリから Remote Control 経由で Notion のKPIデータを取得した様子。ヘッダーの「Interactive session」がRemote Control接続を示している。*

Claude Code はローカルの MCP サーバー経由で Notion API を呼び出し、データベースを読み取り・更新します。スマホからの入力なのに、ローカル PC 上の MCP が動いているので、認証情報の管理も安全です。

### ユースケース2: コンテンツカレンダーに記事を追加

電車の中で記事のアイデアを思いついたとき：

```
コンテンツカレンダーに新しい記事を追加して。
タイトル: 「MCPサーバーの作り方入門」
PF: Zenn記事
カテゴリ: AI活用開発
公開予定日: 来週金曜
```

MCP 経由で Notion のデータベースに直接レコードが追加されます。

### ユースケース3: バグ修正の指示

App Store Connect からクラッシュ通知が来たとき、移動中でも：

```
src/services/revenueCat.ts を読んで、addCustomerInfoListener の
エラーハンドリングを確認して。try/catch が足りない箇所があれば修正して。
```

ローカルのソースコードを読み、修正し、git commit まで実行できます。帰宅後にビルド・提出するだけです。

## 注意点とハマりポイント

### 1. 初回の認証エラー

```
Remote Control is not enabled for your account
```

このエラーが出た場合：
```bash
claude /logout
claude /login
```
ログアウト → 再ログインで解決します。

### 2. API 500エラー

Research Preview のため、サーバーエラーが発生することがあります。少し待って再試行してください。

### 3. セッションのタイムアウト

PCがスリープしたりネットワークが切れると、約10分でセッションがタイムアウトします。PCがオンラインに復帰すれば自動再接続されます。

### 4. 権限の自動承認は無効

Remote Control 中は `--dangerously-skip-permissions` が使えません。ファイル編集やコマンド実行のたびに承認が必要です。これはセキュリティ上の仕様です。

### 5. 同時接続は1つまで

1つの Claude Code インスタンスにつき、リモート接続は1つだけです。

## まとめ

Remote Control の本当の価値は「スマホ対応」ではなく、**ローカル環境の全リソースにどこからでもアクセスできること**です。

特に MCP との組み合わせは強力で：

- **Notion MCP** → 移動中に事業管理
- **GitHub MCP** → PR レビューやイシュー管理
- **Supabase MCP** → データベースの確認・修正
- **Slack MCP** → 通知確認とレスポンス

これらがすべてスマホの Claude アプリから、ローカル PC の認証情報を使って安全に実行できます。

「PCの前にいないと仕事にならない」という制約がなくなるのは、週15時間で副業している身としてはかなりのインパクトです。スキマ時間が全部作業時間に変わります。

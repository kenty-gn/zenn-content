ローカルのClaude Codeセッションを、スマートフォンやブラウザからそのまま操作できる。

それが2026年2月25日にAnthropicが発表した **Claude Code Remote Control** です。VPN設定もポート開放も不要で、iPhoneを手に取るだけで自宅PCのターミナルが使えるようになります。

発表から1週間、実際に使い込んだ上でわかったことを、セットアップから実践的な使い方まで余すところなく解説します。

## Claude Code Remote Control とは

### 一言で言うと

「ローカルで動き続けるClaude Codeセッションに、スマホやブラウザから接続できる機能」です。

重要なのは、**セッションはクラウドに移動しない**という点です。コードもファイルも環境変数も、すべて自分のPC上に留まります。スマートフォンやブラウザはその「窓」として機能するだけです。

```
┌──────────────┐      HTTPS      ┌──────────────┐      HTTPS      ┌──────────────┐
│  iPhone /    │ ◄────────────  │  Anthropic   │  ────────────► │  自宅PC       │
│  ブラウザ     │                │  API (中継)  │                │  Claude Code │
└──────────────┘                └──────────────┘                └──────┬───────┘
                                                                       │
                                                                ローカル環境
                                                                ├── ファイルシステム
                                                                ├── 環境変数(.env)
                                                                ├── MCPサーバー
                                                                └── Git / npm 等
```

通信は全てAnthropicのAPIサーバーを経由し、TLS暗号化されています。自宅PCにインバウンドのポートを開ける必要はありません。

### 従来の方法との比較

Remote Controlが出る前、外出先でローカル環境を操作しようとすると以下のような方法しかありませんでした。

| 方法 | 手間 | セキュリティリスク |
|------|------|-----------------|
| VPN + SSH | 設定が複雑 | ポート開放が必要 |
| ngrok 等のトンネリング | 毎回設定が必要 | URLが変わる・有料 |
| クラウドの開発環境 | コスト・同期の手間 | ローカルファイルが使えない |
| **Claude Code Remote Control** | **QRコードをスキャンするだけ** | **アウトバウンドHTTPSのみ** |

## 前提条件

### 必要なもの

| 項目 | 詳細 |
|------|------|
| Claude Code | v2.1.52 以上 |
| Claudeプラン | Pro ($20/月) または Max ($100〜200/月) |
| OS | macOS / Linux / WSL（Windowsネイティブは非対応） |
| 接続端末 | Claude iOS/Android アプリ または claude.ai/code （ブラウザ） |

### プランについて

2026年2月25日の発表時点では **Maxプラン（$100〜$200/月）限定**のResearch Previewとして公開されました。その後、Proプラン（$20/月）への展開も発表されており、順次ロールアウト中です。

TeamプランとEnterpriseプランは現時点では対象外です。

### Claude Codeのバージョン確認

```bash
claude --version
```

v2.1.52未満の場合はアップデートします。

```bash
npm update -g @anthropic-ai/claude-code
```

## セットアップ手順

### ステップ1: ログイン状態の確認

Claude CodeでProまたはMaxプランのアカウントにログインしている必要があります。

```bash
claude
# Claude Codeが起動したら /login で確認
```

アカウントを確認したい場合は `/logout` してから `/login` で再認証します。

### ステップ2: Remote Controlの起動

起動方法は2通りあります。

**方法A: 新規セッションとして起動（ターミナルから）**

```bash
claude rc
# または
claude remote-control
```

**方法B: 既存セッションに追加する（Claude Code内から）**

すでにClaude Codeのセッションが進行中の場合、会話の途中から接続を追加できます。Claude Codeのプロンプトに入力します。

```
/rc
# または
/remote-control
```

この方法の場合、**会話の履歴がそのまま引き継がれます**。作業の途中で「外出することになった」という場面でも使えます。

### ステップ3: QRコードの表示と接続

起動すると、ターミナルにセッションURLが表示されます。

```
Remote Control session started
Session URL: https://claude.ai/code/session/xxxxxxxxxx

Press [Space] to show QR code
```

スペースキーを押すとQRコードが表示されます。

**iPhoneからの接続手順:**

1. Claude iOSアプリを開く
2. 標準カメラアプリまたはClaudeアプリ内でQRコードをスキャン
3. 自動でセッションに接続される

**ブラウザからの接続手順:**

1. `claude.ai/code` を開く
2. セッション一覧に「Interactive session」として表示されるのをタップ
3. または表示されたURLを直接ブラウザで開く

:::message
セッションに分かりやすい名前をつけておくと、一覧から探しやすくなります。接続前に `/rename` コマンドで名前を変更できます。
:::

### ステップ4: 接続の確認

スマートフォン側のClaude アプリ上部に「Interactive session」という表示が出れば接続成功です。ここから送ったメッセージはローカルのClaude Codeで実行されます。

## 実際の使い方

### モバイルからのコード確認

```
src/components/HomeScreen.tsx を読んで、パフォーマンス上の問題があれば教えて
```

ローカルにあるファイルをそのまま読み込み、分析します。

### 移動中のバグ修正指示

App Store ConnectからクラッシュレポートのメールがiPhoneに届いたとき:

```
さっきRevenueCatのWebhookでエラーが出た。
supabase/functions/revenuecat-webhook/index.ts を読んで、
PRO_EVENTSのハンドリングにtry/catchが足りない箇所があれば修正して
```

ローカルのソースコードを読み、修正し、git commitまで実行できます。帰宅後にビルドするだけです。

### スキマ時間でのコードレビュー

```
最後のコミットとその前を diff で比較して、レビューコメントを日本語で出して
```

### 外出先からの定期タスク実行

```
npm run lint を実行して、エラーがあれば修正して
```

ターミナルコマンドの実行結果がスマートフォンに返ってきます。ただし、コマンド実行の前に承認を求めるダイアログが表示されます（後述のセキュリティ仕様）。

## セキュリティの仕組み

### アーキテクチャ上の安全性

Remote Controlのセキュリティ設計の核心は「**インバウンドポートを一切開かない**」ことです。

ローカルのClaude Codeは、Anthropic APIに対してアウトバウンドのHTTPS接続をポーリングします。外部から直接PCに接続するのではなく、PCが自分から「仕事を取りに行く」設計です。これはファイアウォールの内側にあるPCでも問題なく動作する理由でもあります。

```
[従来のリモートアクセス]
外部 ──インバウンド接続──► PCのポート (リスク)

[Remote Control]
自宅PC ──アウトバウンドHTTPS──► Anthropic API ◄── スマホ
```

通信経路の詳細:

- **暗号化**: 全通信はTLS（api.anthropic.com）経由
- **認証**: 短命の複数のクレデンシャルを使用。各クレデンシャルは単一の用途に限定され、独立して期限切れになる
- **トラフィック**: 通常のClaude API呼び出しと同じドメイン（api.anthropic.com）

### 操作権限の承認

Remote Control中は、ファイルの編集やシェルコマンドの実行のたびに**手動での承認**が必要です。

通常のClaude Codeで使える `--dangerously-skip-permissions` フラグはRemote Controlでは無効化されています。これは意図的なセキュリティ仕様で、リモート操作中に意図しない操作が実行されるリスクを防ぎます。

スマートフォン側に「このコマンドを実行してよいですか？」という確認が表示されるので、内容を確認してから承認します。

### 接続は1セッションにつき1つ

1つのClaude Codeセッションに接続できるリモートデバイスは1つだけです。セキュリティの観点から、複数のデバイスが同時にセッションを操作することはできません。

## 副業エンジニアへの実践的な活用シーン

副業で週15時間しか開発時間を確保できない身として、Remote Controlが特に効いた場面を紹介します。

### 通勤中のコードレビュー

自宅PCをRemote Controlで起動したまま出勤します。電車の中でiPhoneからプルリクのdiffをレビューし、修正指示を出せます。PCに戻ったときにはコードが修正済みです。

```
前回のgit stashの内容を確認して、レビューコメントをまとめて教えて
```

### 外出先からのデプロイ確認

Vercelのデプロイが失敗したとき、外出先からでも原因調査ができます。

```
直近のVercelデプロイログを確認して。もしローカルで再現できるなら
npm run build を実行してエラーを確認して
```

### 深夜の障害対応（寝る前に指示だけ出す）

App Storeのレビューにバグ報告が来た夜、就寝前にスマートフォンから修正指示だけ出しておきます。

```
レビューに「ログインできない」という報告が来た。
src/services/supabase/auth.ts のApple OAuth処理を確認して、
よくある問題がないかチェックして。問題があれば修正案を出して
```

朝起きたら分析結果が残っています。承認が必要な操作は止まっているので、確認してから承認します。

### アイデアメモをコードに変換

移動中にアイデアが浮かんだとき、iPhoneからざっくりした指示を出しておきます。

```
ふくログに「月次レポートをPDFでエクスポートする機能」を追加したい。
まず実装の設計案を考えて、どのファイルを変更する必要があるか教えて
```

帰宅後に設計案を確認し、そのまま実装に進めます。

## 現在の制限事項

Research Previewのため、いくつかの制限があります。

### 機能面の制限

| 制限 | 詳細 |
|------|------|
| 同時接続数 | 1セッションにつき1デバイスのみ |
| セッションタイムアウト | ネットワーク切断から約10分でタイムアウト（PCがオンラインに戻れば自動再接続） |
| 権限スキップ不可 | `--dangerously-skip-permissions` は使用不可 |
| Teamプラン非対応 | 現時点ではProとMaxのみ（Enterpriseも非対応） |

### 安定性

Research Previewのため、稀にAPIの500エラーが発生します。数分待ってから再試行するか、セッションを再起動してください。

### モバイルUIの制限

スマートフォンの小さい画面では、長いコードブロックや複雑なターミナル出力が見づらい場面があります。コードの詳細確認はブラウザ版の `claude.ai/code` の方が見やすいです。

## よくあるエラーと対処法

### Remote Control is not enabled for your account

```bash
claude /logout
claude /login
```

ログアウトして再ログインすると解決することがほとんどです。

### QRコードが表示されない

スペースキーを押してもQRコードが出ない場合は、セッションURLを手動でコピーしてブラウザで開いてください。

### セッションが途切れる

PCのスリープ設定を確認してください。Remote Control中はPCをスリープさせないよう、電源設定を調整することをおすすめします。macOSの場合:

```bash
# Remote Controlを使う間だけスリープを防ぐ
caffeinate -i &
```

作業が終わったら `kill %1` でcaffeinateを停止します。

### スマートフォンからの入力が遅い

ネットワーク経由での通信があるため、入力から応答まで若干のラグがあります。長い指示は一度テキストアプリで書いてからコピペすると快適です。

## Remote Control vs Claude.ai/code (ブラウザ版)

「ブラウザ版のClaude Codeと何が違うの？」という疑問に明確に答えます。

| 機能 | Remote Control | claude.ai/code |
|------|---------------|----------------|
| ローカルファイルアクセス | 可 | 不可 |
| MCPサーバー利用 | 可（ローカル設定を継承） | 不可 |
| 環境変数(.env) | 利用可能 | 不可 |
| Gitコマンド | 実行可能 | 制限あり |
| セットアップ | PCでの起動が必要 | 不要 |
| PC不要 | PCの電源ONが必要 | 完全不要 |

要するに: **PCの電源が入っていて、ローカル環境が必要な作業にはRemote Control。ちょっとした質問や新規プロジェクトの相談にはclaude.ai/code。**

## まとめ

Claude Code Remote Controlは、「PCの前にいないと開発できない」という制約を大幅に緩和します。

特に副業で開発時間が限られているエンジニアにとって、**スキマ時間を全部作業時間に変えられる**インパクトは無視できません。

現時点のResearch Previewでの主な制限は:

- ProとMaxプランのみ（Teamは非対応）
- 同時接続は1デバイスのみ
- コマンド実行には都度承認が必要
- PCの電源ONが前提

これらの制限を理解した上で使えば、開発体験は確実に変わります。特にMCPサーバーを設定している場合、Notionやsupabase、GitHubといった外部サービスへのアクセスもモバイルから可能になります（MCPとの組み合わせについては[こちらの記事](/articles/6b13ff6d716283)で詳しく解説しています）。

ProプランへのロールアウトもAnthropicが明言しているので、まだMaxプランでない方も近いうちに試せるようになるはずです。

## この記事で学べること

- **NDLOCR-Lite**のインストールから使い方まで
- 既存のOCRツール（Google Cloud Vision / Tesseract）との違い
- 個人開発での活用アイデア

## 前提条件

- macOS / Windows / Linux のいずれか
- Python 3.8以上（CLI利用の場合）
- **GPUは不要**（これが最大の特徴）

GUIアプリ版ならPython環境も不要です。

## NDLOCR-Liteとは

2026年2月24日、国立国会図書館（NDL）のNDLラボが公開した**軽量OCRツール**です。

もともと国会図書館には「NDLOCR」という高精度OCRがありましたが、GPU必須でサーバー向けでした。NDLOCR-Liteはその軽量版で、**一般的なノートPCのCPUだけで動作する**ように設計されています。

**スペック**

- GPU不要（ONNX推論、CPUのみで動作）
- 対応OS: Windows 11 / macOS（Apple M4確認済み）/ Linux（Ubuntu 22.04）
- 対応文字: 日本語活字 / 英文活字 / 手書き文字（試験対応）
- 費用: **完全無料・オープンソース**

https://github.com/ndl-lab/ndlocr-lite

## 環境構築

3つのインストール方法があります。好きな方法を選んでください。

### 方法1: pip

```bash
git clone https://github.com/ndl-lab/ndlocr-lite
cd ndlocr-lite
pip install -r requirements.txt
```

インストール後、`src/` ディレクトリから実行します。

```bash
cd src
python3 ocr.py [入力画像パス]
```

### 方法2: uv（推奨）

```bash
git clone https://github.com/ndl-lab/ndlocr-lite
cd ndlocr-lite
uv tool install .
```

この方法だと `ndlocr-lite` がシステムコマンドとして使えるようになります。

```bash
ndlocr-lite [入力画像パス]
```

個人的には `uv` が一番楽です。

### 方法3: GUIアプリ（プログラミング不要）

GitHubのReleasesページからOS別のデスクトップアプリをダウンロードできます。

https://github.com/ndl-lab/ndlocr-lite/releases

マウス操作だけで使えるので、コマンドラインに慣れていない人にもおすすめ。

### おまけ: Webアプリ版

第三者がWebアプリ版も公開しています。インストール不要でブラウザだけで試せます。「まず精度を確認してからインストールしたい」という人はこちらから。

## 使い方

### CLI（コマンドライン）

```bash
# 1枚の画像をOCR
ndlocr-lite input.jpg

# 出力先を指定
ndlocr-lite input.jpg -o output.txt

# PDFを入力（複数ページ対応）
ndlocr-lite document.pdf
```

### GUIアプリ

1. アプリを起動
2. 画像ファイルをドラッグ&ドロップ
3. 「OCR実行」をクリック
4. テキスト結果が表示される

公式の使い方ガイドが丁寧にまとめられています。

https://lab.ndl.go.jp/data_set/ndlocrlite-usage/

## 既存ツールとの比較

「日本語OCR」で個人開発者が選べる選択肢を整理します。

### Google Cloud Vision API

- 精度: 非常に高い
- 日本語: 対応
- コスト: **従量課金**（1,000画像で$1.50〜）
- GPU: 不要（クラウド処理）
- 特徴: API呼び出しが必要、クラウド依存

精度は最高クラスですが、個人開発だと「課金が怖くて気軽に使えない」のが正直なところ。

### Tesseract

- 精度: 英語は高い、**日本語は低い**
- 日本語: 一応対応
- コスト: 無料
- GPU: 不要
- 特徴: 歴史が長い、コミュニティが大きい

無料で使えるのは魅力ですが、日本語の認識精度が厳しい。フォントや画質によっては壊滅的な結果になります。

### NDLOCR（重量版）

- 精度: 最高クラス
- 日本語: 最適化済み
- コスト: 無料
- GPU: **必要**
- 特徴: サーバー向け、大量バッチ処理向き

精度は文句なしですが、GPU必須なのでノートPCでは動きません。

### NDLOCR-Lite

- 精度: 高い（重量版に迫る）
- 日本語: 最適化済み
- コスト: **無料**
- GPU: **不要**
- 特徴: ノートPCで動く、GUI/CLI/Web全対応

**「無料 × GPU不要 × 日本語高精度」**。個人開発者にとってはこれが最適解だと思います。

## 個人開発での活用アイデア

### 1. 紙のメモ → デジタルタスク化

手帳やホワイトボードの写真を撮って、OCRでテキスト化。タスク管理アプリに取り込むパイプラインが作れます。

```
カメラで撮影 → NDLOCR-Lite → テキスト抽出 → タスク管理アプリに登録
```

### 2. 領収書のテキスト化

紙の領収書をスキャン → OCR → 経費管理に登録。確定申告の準備が楽になります。副業エンジニアなら特に重宝するはず。

### 3. モバイルアプリへの組み込み

自分のサーバーにNDLOCR-Liteをデプロイして、APIとして提供するパターン。

```
Expo + カメラ → 画像をサーバーに送信 → NDLOCR-Lite実行 → テキスト結果を返す
```

自前でOCR APIを持てるので、Google Cloud Visionの従量課金から解放されます。

### 4. 古い書籍・技術書のデジタル化

紙の技術書をスキャンしてテキスト化。検索可能なPDFに変換できます。絶版本やPDF版がない書籍で特に有用。

## 注意点

使ってみて気づいた制限事項も書いておきます。

### 手書き文字は「試験対応」

きれいな手書きなら認識しますが、走り書きは厳しい。手書きメモのOCRを主目的にするなら、Google Cloud Visionのほうが精度は高いです。

### 大量バッチ処理はCPUだと遅い

数百ページの本をまとめてOCRするなら、重量版NDLOCRやクラウドサービスのほうが現実的。NDLOCR-Liteは「ちょっとした画像を日常的にOCRする」用途に最適化されています。

### API提供はなし

NDLOCR-Lite自体にはREST APIなどはありません。アプリに組み込みたい場合は、自分でラッパーを書くか、第三者のWebアプリ版を参考にする必要があります。

## まとめ

NDLOCR-Liteは「日本語OCR難民」だった個人開発者にとって救世主的なツールです。

- **無料** — 利用制限なし
- **GPU不要** — ノートPCで動く
- **日本語に最適化** — 国会図書館が作っているので当然
- **3通りの使い方** — GUI / CLI / Web
- **オープンソース** — カスタマイズ可能

国立国会図書館がこういうツールを無償公開してくれるのは本当にありがたい。個人開発者こそ使い倒すべきツールだと思います。

MacのアプリをAIが直接操作する。クリックも、入力も、ブラウザのページ遷移も。

2026年3月23日、Anthropicが「Claude Computer Use」のmacOSリサーチプレビューを公表しました。画面を見て・マウスを動かして・キーボードを叩く、その一連の操作をClaudeが自律的に実行します。この記事では、何ができるのか・何ができないのか・個人開発者がどう活用すべきかを実践的にまとめます。

## Claude Computer Useとは

Claude Computer Useは、ClaudeがmacOS上のGUIアプリケーションを直接操作できる機能です。

従来のAPIやMCPサーバーとは根本的に発想が違います。「API連携がないツールも、人間が操作するのと同じように操作できる」というアプローチです。

| 項目 | 内容 |
|------|------|
| 発表日 | 2026年3月23日 |
| 対応OS | macOSのみ（Windows x64は今後対応予定） |
| 対象プラン | Pro / Maxのみ（Team / Enterpriseは未対応） |
| ステータス | リサーチプレビュー |

## 技術的な仕組み：コネクタ優先 → 画面操作フォールバック

Computer Useの設計で重要なポイントは、**コネクタ優先のフォールバック設計**です。

```
1. Google Calendar / Slack 等の既存コネクタを確認
   ↓ コネクタがない場合
2. 画面操作（マウス・キーボード・スクリーン）にフォールバック
```

つまり、APIやMCPサーバーが存在するツールにはそちらを使い、存在しない場合に初めて画面操作が走ります。これにより「速い処理はAPIで、対応していないツールは画面操作で」という賢い使い分けが自動でされます。

実際の操作は以下の3つのインターフェースを直接制御します。

- **マウス**: クリック、移動
- **キーボード**: テキスト入力、ショートカット
- **スクリーン**: 現在の画面状態の認識

パーミッション制も採用されており、Claudeが初めて触るアプリの操作を始める前に許可を求めます。意図しないアプリの操作を防ぐ設計です。

## できること

### 開発フロー全体の自動化

個人開発者にとって最もインパクトがある活用例です。

```
コード変更 → ブラウザでの動作確認 → PRの作成 → CIのステータス確認
```

この一連の流れを一気通貫で実行できます。

- IDE内での変更を加える
- `git commit` / `git push` を実行する
- PRを提出する
- テストを実行してステータスを確認する
- ブラウザでデプロイ後の動作確認をする

これまでは「Claude Codeがコードを書く → 自分でブラウザを開いて確認する」という手動のブリッジが残っていました。Computer Useはそのブリッジを消します。

### ブラウザ操作

- リンクのクリックとページ遷移
- テキストの入力・検索
- スクロール（※後述の制限あり）
- フォームの入力・送信

### ローカルファイルとレポート作成

Excelやローカルのスプレッドシートファイルを開き、データを集約してレポートにまとめる操作が可能です。APIで繋がっていない社内ツールや業務ソフトでも対応できるのが強みです。

### iPhoneからMacに指示を投げる（Dispatch連携）

Dispatch（モバイルツール）と組み合わせると、iPhoneからMacのClaude Computer Useにタスクを投げることができます。

```
外出先でiPhoneを操作
→ Dispatchでタスクを送信
→ 自宅のMacがComputer Useを実行
```

「外出中に気になったことをiPhoneでメモするように送ったら、Macが調べてドキュメントにまとめておいてくれた」という使い方が現実になります。

## できないこと・制限事項

リサーチプレビュー段階のため、制限は正直に把握しておく必要があります。

| 制限 | 詳細 |
|------|------|
| OS | macOSのみ。Windowsは未対応（x64は今後対応予定） |
| プラン | Pro / Maxのみ。Team / Enterprise は未対応 |
| スクロール・ドラッグ・ズーム | まだ苦手。精度が低い |
| 処理速度 | テキスト処理より遅く、エラーが出やすい |
| 金融・トレーディング | デフォルトで制限あり |
| 機密データ | Anthropic自身が「使わないで」と推奨 |

特に「**画面操作はテキスト処理より遅くてエラーが出やすい**」という点は重要です。APIで済む処理はAPIを使い、Computer Useは「他に方法がない場合の最終手段」として使う設計が推奨されています。前述のコネクタ優先フォールバック設計はこのため存在します。

## Remote Controlとの違い

2026年2月に発表されたRemote Controlと混同しやすいので整理します。

| 機能 | Remote Control | Computer Use |
|------|---------------|--------------|
| 発表 | 2026年2月 | 2026年3月 |
| できること | スマホ/ブラウザからClaude Codeセッションを操作 | GUIアプリを直接操作 |
| 対象 | Claude Codeのターミナルセッション | macOS上の任意のアプリ |
| 用途 | コーディング作業のリモート管理 | GUIを含む全般的な操作の自動化 |

Remote ControlはあくまでClaude Codeのセッションを遠隔操作する機能です。Computer Useはその発展形として、GUIアプリを含むすべての画面操作を自律的に実行できます。

「ターミナルの外に出て、GUI世界も操作できるようになった」というイメージです。

## 安全対策

自律的にマウスとキーボードを操作するとなると、安全性が気になります。Anthropicは以下の対策を実装しています。

**パーミッション制**
Claudeが初めて操作するアプリには、事前に許可を求めます。意図しないアプリへのアクセスを防ぎます。

**いつでも停止可能**
操作中にユーザーが任意のタイミングで停止できます。暴走しても即座に止められます。

**プロンプトインジェクション対策**
Webページや外部コンテンツにClaudeの行動を書き換えるような指示が含まれていても、それを実行しないための対策が施されています。

**機密データは使わない**
これはAnthropicの公式推奨事項です。Computer Useを通じた操作ログが学習に使われる可能性などを考えると、個人情報・認証情報・機密ファイルを扱う操作には使わないのが安全です。

## 個人開発者の実践的な活用シナリオ

### シナリオ1: 開発→確認→デプロイの一気通貫

```
Claude Codeでコードを書く
→ Computer UseでExpoo Go / シミュレーターを起動して動作確認
→ ブラウザでSupabase Dashboardにアクセスしてデータを確認
→ git push → Vercelのデプロイ状況をブラウザで確認
```

この全工程をClaudeが自律的に実行します。自分はコーヒーを飲みながら結果を待つだけです。

### シナリオ2: APIがない業務ツールの自動化

クラウド会計ソフト、古い業務システム、API連携に対応していないSaaSなど、「APIがないから自動化できない」と諦めていたツールも、画面操作で自動化できます。

```
「freeeで先月の収入・経費をまとめてCSVでエクスポートして」
→ freeeをブラウザで開く → 期間を選択 → CSVダウンロード
```

### シナリオ3: iPhoneからの非同期タスク投入

```
外出中にiPhoneでメモ
→ Dispatch経由でMacに送信
→ Computer UseがMacで調査・作業を実行
→ 帰宅したら結果が揃っている
```

「Macを持ち歩かなくても、iPhoneだけで作業をキューに積める」という体験は個人開発者のワークフローを大きく変えます。

## 試す前のチェックリスト

Computer Useを使い始める前に確認しておくことをまとめました。

- [ ] Pro または Max プランに加入しているか
- [ ] macOSを使っているか
- [ ] 操作させたいアプリのAPI/コネクタが存在しないか確認したか
- [ ] 機密情報・個人情報を扱う操作でないか確認したか
- [ ] 金融・トレーディングプラットフォームの操作は含まれないか
- [ ] 操作をいつでも停止できる状態にしているか

## まとめ

Claude Computer Useは「AIがコードを書く」から「AIがPCを操作する」への大きな飛躍です。

まだリサーチプレビューなので、スクロール・ドラッグの苦手さや速度の問題はあります。ただし「コネクタ優先・画面操作フォールバック」という設計思想は理にかなっており、実用性は着実に上がっていくはずです。

個人開発者として最初に試すべきは「API連携のないツールを使った単純作業の自動化」です。処理が成功してもエラーになっても、その体験がComputer Useへの解像度を上げる一番の近道になります。
